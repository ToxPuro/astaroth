## Overview

C++ implementation: `samples/tfm-mpi/tfm.cc`

DSL implementation: `samples/tfm/mhd/mhd.ac` (same physics, different execution approach)

Config file: `samples/tfm/mhd/mhd.ini`

A **MPI-accelerated Test Field Method (TFM) MHD simulator** with a hand-coded C++ simulation pipeline (`rev::Grid`). The TFM solves for the evolution of four test field pairs (a11, a12, a21, a22) coupled to MHD velocity/density, computing mean-field coefficients (alpha, eta tensors) from the cross-helicity and electromotive force (EMF) statistics. Unlike the DSL-based `samples/tfm/` variant, this version implements the full simulation loop, RK3 substep scheduling, and halo exchange logic directly in C++ rather than relying on the DSL task graph builder.

**Note:** `samples/tfm-mpi/` is the MPI-accelerated executable version; `samples/tfm/` is the DSL-based reference implementation.

### TFM Pipeline (`rev::Grid::tfm_pipeline`)

The simulation uses **RK3 time integration** split across 3 substeps with **segmented computation** (halo vs inner domain) and **two independent halo exchange batches**:

```
For each RK3 substep (0, 1, 2):

  1. Update dt (from MPI_MAX reduction of uumax/Alfven speeds) and forcing
  2. Outer segments (halo region):
     a. wait_hydro_he(input)         → complete previous hydro halo exchange
     b. compute_outer(hydro_kernels, substep)  → MHD update on outer region
     c. launch_hydro_he(output)      → async halo exchange for hydro fields
     d. wait_reduce_xy_averages()    → XY-averaged profile reductions
     e. wait_tfm_he(input)           → complete previous TFM halo exchange
     f. compute_outer(tfm_kernels, substep)      → TFM test field update on outer region
     g. launch_tfm_he(output)        → async halo exchange for TFM fields
  3. Inner segments (computational domain):
     a. compute_inner(hydro_kernels, substep)
     b. compute_inner(tfm_kernels, substep)
     c. swap_buffers()               → swap input/output vertex buffers
  4. if substep == 2: launch_dtmax_reduce()   → prepare next dt
  5. launch_reduce_xy_averages()     → start next XY-average reduction

Finalize: wait all halo exchanges and reductions
```

### Timestep Calculation

Timestep is computed per-step at `tfm.cc:316-354` in `calc_timestep()`:

```
uu_dt = cdt * dsmin / (|uumax| + sqrt(cs²_sound + vAmax²))
visc_dt = cdtv * dsmin² / max(nu_visc, eta, eta_tfm, gamma*chi)
dt = min(uu_dt, visc_dt)
```

Where:
- `cdt = 0.4` (Courant number for advection)
- `cdtv = 0.25` (Courant number for diffusion)
- `vAmax` is optional (only when `TFM_DT_INCLUDE_ALFVEN` is defined)
- `shock_max` is **never** used (warning shown once at `tfm.cc:1568`)

### Halos and XY Averages

Two independent halo exchange batches:
- **Hydro fields**: `VTXBUF_LNRHO, VTXBUF_UUX, VTXBUF_UUY, VTXBUF_UUZ` (4 fields)
- **TFM fields**: TF_a11_x/y/z, TF_a12_x/y/z, TF_a21_x/y/z, TF_a22_x/y/z (12 fields)

XY averages are computed via `acDeviceReduceXYAverages()` on the device, then allreduced across the XY-neighbor communicator (`m_xy_neighbors`) using `twoway_buffered_iallreduce` with **host buffers** (device-buffered IALLREDUCE is disabled for Mahti compatibility).

## Notable Observations

1. **Custom TFM pipeline**: Unlike `samples/tfm/` which uses the DSL task graph builder, this version hand-codes the entire RK3 pipeline with explicit segmented kernel launches. The DSL (`mhd.ac`) is referenced but the execution is direct C++.

2. **Segmented computation**: Halos and inner domain are computed separately via `acDeviceLoadIntUniform(AC_exclude_inner, 0/1)` — the kernels check this flag to skip outer or inner regions.

3. **LUMI topology mapping**: GPU selection is hardcoded for LUMI's 8-GPU node topology (`6, 7, 0, 1, 2, 3, 4, 5`) as noted at `tfm.cc:1210`. The `select_device()` function warns about this at runtime.

4. **Dry-run initialization**: The `rev::Grid` constructor runs a 3-step dry-run of `tfm_pipeline()` followed by a full reset to validate the pipeline before the actual simulation starts (`tfm.cc:1336-1341`).

5. **Forcing sequence**: A new forcing vector is generated at the start of every iteration. The RNG seed is fixed at `19937123`, so re-running produces the same forcing sequence. Forcing is NOT saved to disk.

6. **Test field reset**: Test fields are periodically reset to zero at intervals specified by `AC_simulation_reset_test_field_interval` (step-based) and `AC_simulation_reset_test_field_time_interval` (time-based), with the time-based check being an additional guard.

7. **Profile I/O uses sub-communicators**: `write_profiles_to_disk()` splits into XY-plane sub-communicators (`tfm.cc:418-421`) so only ranks with `coords_x=0 && coords_y=0` write profiles, reducing I/O contention.

8. **Mahti workaround**: CUDA-aware MPI IO is not supported on Mahti; `Snapshot::read()` and `Snapshot::write()` explicitly copy data through host buffers before I/O.

9. **Async profile writes are conditional**: `AC_WRITE_ASYNC_PROFILES` enables `std::future`-based async profile writing (`tfm.cc:455-555`). The async writer packs the profile to a host buffer, then launches an async task to write directly to file at the correct offset.

10. **Alfven speed reduction**: When `TFM_DT_INCLUDE_ALFVEN` is defined, the timestep calculation includes Alfven speeds for all 4 test field pairs (`vAmax11/12/21/22`) via `RTYPE_ALFVEN_MAX` reduction (`tfm.cc:1478-1518`).

11. **SOCA mode**: Setting `AC_TFM_SOCA_enabled = 1` in the INI file enables the Single-Order Closure Approximation, simplifying the TFM equations.

12. **Magnetic Laplace diffusion**: Setting `AC_TFM_magnetic_laplace_diff = 1` switches from the standard `-eta*j` diffusion to `+∇²a` for the test field vector potential update.

13. **Production run advice**: The README recommends maximizing `snapshot_output_interval` and `profile_output_interval` (both are synchronous I/O) and disabling log messages by setting `ACM_LOG_LEVEL = ACM_LOG_LEVEL_ERROR`.

14. **Debug outputs**: Setting `LOO=1` enables vorticity (curl(UU)) output to `VTXBUF_OOX/Y/Z` and helicity (`dot(uu, curl(uu))`) to `VTXBUF_UU_DOT_OO`. Roberts flow mode (`LHYDRO=0`) is available as a preliminary time-independent test.

15. **Benchmark mode**: Writes per-rank timing CSVs (`bm-tfm-mpi-<jobid>-<rank>-<name>.csv`) with per-sample nanosecond counts, then optionally profiles with `cudaProfilerStart()`/`cudaProfilerStop()`.

16. **Python coefficient analysis**: The `tfm_coeffs.py`, `construct_coeffs.py`, and `tfmcoeffs.py` scripts compute alpha and eta tensors from EMF profiles using trigonometric decomposition:
    ```
    α11 = (cos(z)*EMF11x + sin(z)*EMF12x) / B_ampl
    η12 = -(sin(z)*EMF11x - cos(z)*EMF12x) / (kz * B_ampl)
    ```

17. **Rank ordering output**: `write_rank_ordering_to_disk()` writes `rank-to-subdomain-mapping-<jobid>.txt` showing MPI rank to subdomain assignment.

18. **Deprecated slice I/O**: `write_slices_to_disk()` exists but is not called from the main simulation. The README marks `.slice` files as deprecated with no effect.

19. **Commented-out code**: Several code paths are commented out, including `get_local_mesh_info()` (deprecated domain decomposition), the old timestep calculation, the old IALLREDUCE with device buffers, and various profiling sections.

20. **Hardcoded parameters in scripts**: `tfm_coeffs.py` has `kz=1.0, Bampl=1.0, kf=5.0, eta=5.0` hardcoded. The `results_pm.txt` file contains production run sweeps of magnetic Prandtl number (Pm: 0.01-10) against forcing wavenumber (kf: 1.5, 5).

21. **All fields initialized to zero except B11/B12/B21/B22 profiles**: `init_tfm_profiles()` zeros all profiles then loads cosine/sine initial conditions for the four B-field profiles only (`tfm.cc:203-239`).

22. **Snapshots use dual-buffer rotation**: Restart uses a 2-snapshot buffer (`nsnapshots=2` at `tfm.cc:1677`), rotating between current and previous. State file `simulation_state.txt` tracks the latest valid snapshot number.

23. **Final output**: After the simulation loop ends, final monolithic snapshots are always written regardless of `snapshot_output_interval` (`tfm.cc:1784-1786`), ensuring at least one complete output.
